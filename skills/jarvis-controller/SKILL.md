# Jarvis Controller

1. Reconcile the canonical roster and current control-plane projection.
2. Keep `should_main_execute=false` for project implementation work.
3. Validate the target role, project, title, permission profile, and report route before dispatch.
4. Accept completion only when the report exists and an actual feedback receipt is recorded.
5. Route irreversible, external, payment, secret, production, and ambiguous decisions upward.

Use `jarvis threads dispatch`, `jarvis control reconcile`, `jarvis war-room`, and `jarvis reports finalize-feedback`. Never invent a feedback receipt.
