# 第一层：自动批量采集（20 分钟，6 步）

## S1：GitHub 项目扫描

**获取方式**：GitHub REST API（无需鉴权）

**脚本**：`scripts/bootstrap.mjs`

**采集内容**：
- 所有公开仓库的名称、描述、语言、Topics、更新时间
- 每个仓库的 README（前 300 字）
- 顶层目录结构（前 20 项）
- 自动分类：active（30天内更新）、paused（30~180天）、archived（>180天）、idea（无代码）

**能揭示什么**：
- 技术栈全景图
- 兴趣域分布（MC Mod、AI工具、课程项目、创业项目）
- 活跃节奏（哪些时期项目创建密集）
- 项目的"保质期"——灵感型 vs 持续投入型

**输出文件**：`knowledge-base/projects/*.md` + `projects/README.md` + 向量库

---

## S2：电脑系统扫描

**获取方式**：`winget list` + 文件系统遍历

**采集内容**：
- 已安装软件列表（开发工具、AI 工具、创作工具、PCB 设计工具）
- D:/ 盘下所有项目目录
- 最近修改的 Markdown 文件
- 下载文件夹内容（电子书、文档、安装包）
- 桌面文件
- Claude Code 项目目录（`~/.claude/projects/`）

**能揭示什么**：
- 实际使用的工具栈（不只是 GitHub 上标的）
- 本地活跃项目（比 GitHub 更准确——很多项目不在 git 里）
- 阅读书单和兴趣方向（从下载文件推断）
- 硬件方向投入（PCB 工具 = AI 学习终端硬件）
- Claude Code 使用频率（每个项目一个会话目录）

**输出文件**：`knowledge-base/computer-scan-report.md`

---

## S3：浏览器历史分析

**获取方式**：复制 Edge History SQLite 数据库，SQL 查询

**前置条件**：关闭 Edge 浏览器（避免文件锁定），复制到临时目录

**采集内容**：
- 最近 150 条 URL 访问记录（标题 + 时间 + 访问次数）
- 域名访问量排名 Top 30
- 近期搜索关键词

**能揭示什么**：
- 信息源结构（GitHub、知乎、B站、Z-Library、抖音的比例）
- 竞品研究痕迹（慧中标、赵逍遥网站）
- 商业调研模式（技术入股、AI 课程设计等搜索）
- 工具焦虑（Codex 反复研究、Cursor 对比）
- 对照模板（赵逍遥——"一人即公司·系统成复利"）

**输出文件**：`knowledge-base/browser-history-analysis.md`

---

## S4：搜索关键词提取

**获取方式**：同一份 Edge History SQLite，提取 `?q=` 参数的 URL

**采集内容**：
- 210 个唯一搜索关键词及搜索次数
- 按时间排列的搜索趋势

**能揭示什么**：
- 真实的时间线（搜索行为不会说谎）
- 创业各阶段的关注焦点（四月→教育调研，五月→公司注册，六月→Codex）
- 硬件调研的深度（查了具体芯片型号 RK3568）
- 教辅材料搜集（物理、数学讲义）
- 另外发现的公司名（"水起云程有限公司"）

**输出文件**：`knowledge-base/search-keywords.json`

---

## S5：Git 提交历史

**获取方式**：遍历本地所有 `.git` 目录，`git log --since`

**采集内容**：
- 每个本地仓库的提交数、首次提交日期、最近提交日期
- 月度提交热力图

**能揭示什么**：
- 真实的开发节奏（不是日记里说的，是 Git 记录的）
- 工作模式：单会话完成型（16 commits 同一天）vs 持续迭代型
- 哪个项目在什么时候有真实投入
- Practising-diary 的停止时间（2026-01-05 → 记录方式从手动 Markdown 变成 AI 对话）

**输出文件**：`knowledge-base/github-commit-analysis.json`

---

## S6：消费记录分析

**获取方式**：用户在支付宝/微信支付导出 CSV/XLSX，放在微信文件目录

**前置条件**：用户手动导出（手机操作，2 分钟）

**采集内容**：
- 月度消费金额和笔数
- 按商户/产品分类汇总
- 大额支出明细
- 退款记录

**能揭示什么**：
- 真实的消费结构（钱不会说谎）
- AI 工具投资意愿和金额
- 创业投资爆发点（五月 ¥11,034）
- 收入来源和稳定性（主要是家人转账）
- 休学时间点（校园卡最后一次充值 1月10日）
- 消费模式：功能型，几乎无冲动消费

**输出文件**：`knowledge-base/consumption-analysis.md`

# 第二层：需要工具辅助（20 分钟，3 步）

## S7：豆包对话导出

**获取方式**：Playwright + Edge 持久化 Profile + 无限滚动 + DOM 抓取

**脚本**：`scripts/export-doubao.mjs`

**前置条件**：关闭 Edge，安装 Playwright (`npm install playwright`)

**采集内容**：
- 侧边栏无限滚动，加载所有对话（327 个）
- 逐个导航到对话页面，提取 `document.body.innerText`
- 保存为 JSON

**技术难点**：
- Edge App-Bound Encryption 阻止 cookies 解密 → 不能用 API，必须用浏览器自动化
- `networkidle` 超时 → 改用 `domcontentloaded` + 主动 `waitForTimeout`
- API 返回 HTML 而非 JSON → 不走 API，直接导航 + DOM 抓取

**能揭示什么**：
- 这是了解主人最深的数据源（327 个对话，451 万字）
- 商业模式调研（Codex、AI 课、教培、跨境电商）
- 会议录音转录习惯（6 个"深空说明会"对话 250 万字）
- 自我剖析对话（147 个涉及人格/心理/认知主题）
- 决策模式：不要答案只要框架
- 情感处理方式：极端情绪时对 AI 说，不对真人说
- 反迎合提示词——人格签名级的发现

**输出文件**：`knowledge-base/doubao-export.json`

---

## S8：Claude Code 项目目录扫描

**获取方式**：读取 `~/.claude/projects/` 目录列表

**采集内容**：
- 22 个项目目录名称
- 每个目录的会话历史文件

**能揭示什么**：
- 实际使用 Claude Code 的项目（比 GitHub 更真实）
- 项目越多 = AI 依赖度越高
- 目录名编码模式（短名 + 分类后缀）

**输出文件**：合并在 `computer-scan-report.md` 中

---

## S9：浏览器书签和扩展

**获取方式**：读取 Edge `Bookmarks` JSON 文件

**采集内容**：
- 书签栏结构
- 安装的浏览器扩展

**能揭示什么**：
- 工具偏好（沉浸式翻译、广告拦截器等）
- 收藏的信息源

# 第三层：人工 + AI 深度分析（50 分钟，2 步）

## S10：日记深度阅读

**获取方式**：clone Practising-diary 仓库，Read 所有 .md 文件

**核心文件**：
- `我的人生指南.md` — 十条人生准则
- `2025年终总结2.md` — 年度回顾 + 核心叙事
- `如何解决焦虑.md` — 情感处理模式
- `deepseek劝学.md` — AI 对学习的态度
- `读《当下的力量》有感.md` — 灵性/心理学偏好
- 每月每日的番茄记录

**能揭示什么**：
- 价值观核心（"赎身"）
- 情感创伤（化竞失败）
- 效率系统的发现与断裂（番茄工作法的起伏曲线）
- 阅读品味（大明王朝、红楼梦、毛选、资本论）
- 身份认同的演变（"生活的体验者和表达者"）

**输出文件**：`knowledge-base/about-me.md`

---

## S11：直接询问主人

**获取方式**：对话，Jarvis 提问，主人回答

**关键问题模板**：
1. "你最近一年发生了哪些重要的事？按照时间顺序告诉我"
2. "你最重要的三个目标是什么？最困扰你的三件事是什么？"
3. "你和哪些人有重要的合作或关系？他们各自扮演什么角色？"
4. "你的钱主要花在哪里？收入从哪里来？"
5. "你希望我了解你到什么程度？有什么是你不希望我碰的？"

**能揭示什么**：
- 数据无法覆盖的事件（心声、打算、人际关系）
- 当前最紧迫的决策
- 隐私边界

# 第四层：高隐私数据（可选，15 分钟，1 步）

## S12：微信聊天记录

**获取方式**：开源工具 [WeChatMsg](https://github.com/LC044/WeChatMsg) 或官方导出

**数据位置**：`C:/Users/<user>/Documents/xwechat_files/<wxid>/db_storage/message/message_0.db`

**当前状态**：数据库完整（10.4MB），key_info.db 可读，但 SQLCipher 加密

**建议导出范围**：
- 不需要全量
- 优先：堂姐老板、班主任、合作伙伴
- 可选：重要群聊（创业相关）

**能揭示什么**：
- 人际关系网络图
- 沟通风格（正式/随意、主动/被动、简短/长篇）
- 关键商业对话的决策时间线
- 与 AI 对话的表达差异对比

# 实施顺序建议

```
第一轮（当天完成）：
  S1 GitHub 扫描 ← 最先做，零门槛
  S2 系统扫描
  S3 浏览器历史
  S6 消费记录 ← 需要用户导出，但操作简单
  产出：初步技术画像 + 行为模式

第二轮（第二天）：
  S4 搜索关键词
  S5 Git 提交历史
  S7 豆包对话导出 ← 最重但价值最高
  S10 日记深度阅读
  S11 直接询问
  产出：深度人格画像 + 完整报告

第三轮（可选）：
  S12 微信聊天记录 ← 需要用户心理准备

每轮结束后更新 comprehensive-personal-report.md
```

# 踩过的坑

| 坑 | 教训 |
|-----|------|
| Edge 文件被锁定 | 必须先 `taskkill //F //IM msedge.exe` |
| Playwright `networkidle` 超时 | 改用 `domcontentloaded` + 主动等待 |
| 豆包 API 返回 HTML | SSR 页面，走 DOM 抓取不走 API |
| Edge cookies App-Bound Encryption | 放弃解密，用 Playwright 持久化 context |
| 侧边栏懒加载 | 必须无限滚动了才能拿到全部对话 |
| CSV GBK 编码 | 用 `iconv-lite` 解码 |
| 支付宝金额 NaN | 金额在 `cols[6]` 不是 `cols[5]` |
| Playwright `channel: 'msedge'` 不兼容 | 用 `executablePath` 直接指定 Edge 路径 |
