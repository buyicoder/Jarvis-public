# Jarvis 自我迭代设计

## 核心原则

Jarvis 的迭代不能依赖"人类程序员主动改进"。必须有一个**数据驱动的反馈闭环**：每次使用 → 收集信号 → 识别瓶颈 → 自动或半自动改进。

## 一、当前瓶颈诊断

| 瓶颈 | 严重度 | 根因 |
|------|--------|------|
| TF-IDF 词表固定，新词汇丢失 | 🔴 高 | 词表在 bootstrap 时构建，之后不更新 |
| 会话记忆不持久 | 🔴 高 | 跨会话上下文仅依赖 daily 文件 |
| 无主动感知 | 🟡 中 | 只能被动等用户打开 |
| 无自我评估 | 🟡 中 | 没有衡量"我做得好不好"的指标 |
| 启动加载固定 | 🟡 中 | 每次都加载相同内容，不管用户当前状态 |
| 检索精度退化 | 🟡 中 | 新内容增多后，旧词表覆盖率持续下降 |

## 二、反馈闭环设计

```
用户打开 Jarvis
      ↓
用户提问 / 下达任务
      ↓
Jarvis 执行（搜索、分析、回答）
      ↓
用户反馈信号收集：
  - 对话长度（聊得越久 = 越有用）
  - 用户是否执行了建议（从 daily 验证）
  - 用户是否回了"ok"、"好的"（隐性满意）
  - 用户是否回来同一话题（持续性）
  - 搜索结果的 relevance（用户有没有说"不是这个"）
      ↓
每日索引时自动评估：
  - 今天新增了多少 chunks？
  - 搜索测试查询的 relevancy score？
  - 词表覆盖率是否下降（新词/总词比）？
      ↓
识别瓶颈 → 触发改进
```

## 三、改进维度

### 1. 知识获取（Know More）

| 改进 | 触发条件 | 实现方式 |
|------|----------|----------|
| 词表自动扩展 | 新增 chunks 中新词占比 > 10% | 重新 buildVocabulary，upsert 增量词条 |
| 从对话中提取事实 | 用户说"我最近在……" 或 "我喜欢……" | daily 总结时自动提取 → 写入 core/ 对应文件 |
| 项目状态自动更新 | daily 中提到项目名 | index-daily 自动更新项目文件的"当前进度" |
| 新数据源接入 | 用户说"帮我分析一下 X" | 自动检测 X 的类型（文件/URL/API），生成采集脚本 |

### 2. 检索精度（Find Better）

| 改进 | 触发条件 | 实现方式 |
|------|----------|----------|
| 替换 TF-IDF 为 embedding | 手动触发（一次性） | 安装 node-llama-cpp + bge-small-zh GGUF |
| 搜索结果排序优化 | 用户说"不是这个" | 标记负样本，调整相似度权重 |
| 时间衰减 | 自动 | 搜索结果乘上时间衰减因子（最近30天 ×1，30-90天 ×0.8，>90天 ×0.5） |
| 上下文感知搜索 | 检测当前项目 | 搜索加上当前项目名作为 filter |

### 3. 交互质量（Talk Better）

| 改进 | 触发条件 | 实现方式 |
|------|----------|----------|
| 风格微调 | 用户说"太啰嗦"/"简洁点" | 在 CLAUDE.md 中累积风格偏好 |
| 情绪感知 | 用户短回复、消极用词 | 调整回答长度和语气（不追问、不逼） |
| 时间感知 | 用户说"几点"、"天晚了" | 主动 check 系统时间 |
| 记忆引用 | 提到过去的事 | 在回答中引用具体日期和来源 |

### 4. 主动性（Do More）

| 改进 | 触发条件 | 实现方式 |
|------|----------|----------|
| Cron 提醒 | 手动设置 | 睡觉/起床/每周复盘 |
| 项目停滞预警 | 某个项目 7 天未在 daily 出现 | 下次会话时提醒 |
| 待办过期提醒 | daily 中未勾选的待办 > 3 天 | 启动时主动报告 |
| 习惯追踪 | daily frontmatter 中 sleep/wake 字段 | 每周统计睡眠趋势，偏离时提醒 |

## 四、自我评估指标

每次 index-daily 运行时自动计算：

```
relevancy_score: 对 5 个测试查询（"modfactory"、"创业"、"合作"、"苏瑾"、"物理补课"）执行搜索，计算搜索结果中正确命中来源的比例
coverage_score: 新增 chunks 中不在词表内的词汇占比。超过 10% → 触发重建
freshness_score: 最近 7 天的 chunks 占总 chunks 的比例
usage_score: 今天新增的 daily 内容长度（间接衡量 Jarvis 的使用深度）
```

结果写入 `index/_health.json`：

```json
{
  "date": "2026-06-20",
  "relevancy": 0.3,
  "coverage": 0.45,
  "freshness": 0.08,
  "usage": 4521,
  "warnings": ["coverage high — consider rebuilding vocabulary", "relevancy low — TF-IDF degradation"]
}
```

## 五、实施路线

| 阶段 | 改进 | 触发方式 |
|------|------|----------|
| **本周** | 词表自动扩展（index-daily 检测覆盖率） | 自动 |
| **本周** | 时间衰减（搜索结果加权） | 自动 |
| **本周** | _health.json 监控 | 自动 |
| **下周** | 替换 TF-IDF → bge-small-zh embedding | 手动触发 |
| **下周** | 项目停滞预警 | 自动 |
| **随时** | 风格偏好累积 | 用户反馈驱动 |

## 六、升级路径

```
Jarvis v1（今天）
  ├── 分层记忆 ✅
  ├── 自动索引 ✅
  ├── Cron 提醒 ✅
  ├── 词表自动扩展 ⬜
  ├── 时间衰减 ⬜
  └── 健康监控 ⬜

Jarvis v2（下周）
  ├── 真正 embedding
  ├── 项目停滞预警
  └── 搜索结果大幅提升

Jarvis v3（下月）
  ├── 主动推送（不打开也能收到提醒）
  ├── 多模态记忆（图片、截图、录音）
  └── 关联图谱可视化
```
